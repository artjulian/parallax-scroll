Moving the general background

This function works exactly like Richard Shepherd's Parallax Scrolling Tutorial example. The background changes position as you scroll. The main cause for your parallax effect.

  • data-type: background
  • data-speed: integer (higher is slower, negative is scrolling with the scroll)

Vertically moving objects

To move objects vertically, the objects are set as backgrounds after which they get a fixed position for performance reasons. The variables below can be used to control the animation.

  • data-type: sprite-vertical
  • data-start: the position you'd like the background to start in pixels (integer)
  • data-end: the position you'd like the background to end in pixels integer(if left empty, it will scroll indefinitely)
  • data-xPos: the x-axis position in any css bg-position unit
  • data-speed: scroll-speed integer (positive integer for scroll animation up, negative for scroll animation down)

Horizontally moving objects

Same as the vertically moving objects, the objects are backgrounds that get a fixed position. The two main differences: start is in percentages en yPos is pixels.

  • data-type: sprite-horizontal
  • data-start: the position you'd like the background to start in percentages (integer)
  • data-end: the position you'd like the background to end in percentages integer(if left empty, it will move indefinitely)
  • data-yPos: the y-axis position in any css bg-position unit
  • data-speed: scroll-speed integer (positive integer for scroll animation up, negative for scroll animation down)

Vertical sticky blocks

In order for block elements (derived from display:block) to move smoothly they need to have a fixed position (mainly for performance reasons). However, when applying a fixed position to an element, all other elements that depend on this element for their relative position will move. Therefore, we clone this element and hide the original to create the animation, then switch it back when the element stops.

  • data-type: vertical-block
  • data-start: the position you'd like the block to start in pixels (integer)
  • data-end: the position you'd like the block to end in pixels integer(if left empty, it will move indefinitely)
  • data-speed: scroll-speed integer (positive integer for scroll animation up, negative for scroll animation down)

Horizontal sticky blocks

This block works the same way as the vertically moving one, with cloning and all.

  • data-type: vertical-block
  • data-start: the position you'd like the block to start in pixels (integer)
  • data-end: the position you'd like the block to end in pixels integer(if left empty, it will move indefinitely)
  • data-speed: scroll-speed integer (positive integer for scroll animation up, negative for scroll animation down)
